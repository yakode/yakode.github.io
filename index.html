0. 前言

現役筆電感覺快到生命的盡頭了，為了換筆電後可以快速把 Hexo 恢復，就把查到的資料整理起來囉!

這只是方便我重新架 Hexo 的整理，如果你是被搜尋引擎帶到這且期待詳盡教學的生物，請直接參考最下方的資料來源，有大神的文章!

1. Hexo環境

  1. 安裝 Hexo
  2. 初始化

2. 發表文章

  1. 產生 .md 檔案
  2. 修改內文

    /blog/source/_posts/<nameOfPost>.md

    修改 <nameOfPost>.md 的內容即可

3. 更換主題(Next)

  1. clone 檔案
  2. 更改主題

    修改 /blog/_config.yml
  3. 調整主題
    1. 語言

      修改 /blog/_config.yml 以及 /blog/theme/next/_config.yml
    2. Scheme 設定

      修改/blog/theme/next/_config.yml

4. 修改基本資料

修改 /blog/_config.yml

5. 修改 sidebar

  1. 加上個人個片

    修改 /blog/theme/next/_config.yml

    在 url: 左方加上圖片位置，若要使用本地圖片，檔案要放在 /blog/source 底下

  2. 社群網站

    修改 /blog/theme/next/_config.yml

    修改內容後，取消註解

  3. 加上 tags/categories 分頁

    1. 在 sidebar 顯示圖標

      修改 /blog/_config.yml

      把 categories 取消註解

    2. 建立新頁面

    3. 修改 /blog/source/categories/index.md

    4. 為文章分類

      如果 new post 時沒自動跳出「categories:」，就自己加上，也會有效果

    5. tags

      方法相同

6. 丟上github

  1. 在 github create repository
  2. 安装 deploy git 插件
  3. 修改/blog/_config.yml
  4. deploy

    產生靜態網頁後 deploy
